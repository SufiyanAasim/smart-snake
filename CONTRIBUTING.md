# Contributing to Smart Snake Game

Thank you for choosing to contribute! Here are the guidelines to get started.

## Branch Naming Policy
We follow structured branch naming. Prefix your branches:
- `feature/` for new functionality
- `bugfix/` or `fix/` for bug fixes
- `refactor/` for code refactoring
- `test/` for unit/integration tests
- `docs/` for documentation
- `experimental/` for AI research

Example: `feature/astar-optimizer`

## Commit Message Convention
We adhere to Conventional Commits:
- `feat:` for new features
- `fix:` for bug fixes
- `docs:` for documentation modifications
- `refactor:` for code restructuring
- `perf:` for performance optimizations
- `style:` for formatting/visual edits

Example: `feat(ai): optimize A* heuristic calculations`

## Pull Request Process
1. Fork the repository and create your branch from `develop`.
2. Ensure your changes compile and pass manual checks.
3. Submit a Pull Request detailing the changes, testing, and related issues.
