# Smart Snake — v5.0.0

> *Codename: Glowserpent*

---

## 📖 Overview

**Glowserpent** is the UI and rendering refinement release. It fixes the score-zero bug, introduces a render-to-texture canvas pipeline that supports arbitrary window resizing, adds borderless fullscreen mode, and introduces a custom dark-themed name-input dialog on Game Over.

Building on the relational persistence of **Vaultserpent**, this release focuses on correctness and polish — every pixel now scales cleanly, and every session's name is prompted in a styled, on-brand dialog.

---

## 🎯 Objectives

* Fix the off-by-one score bug so fresh games always start at `0`.
* Scale the game canvas properly onto a fixed 800 × 600 render target to support window resizing.
* Add borderless fullscreen mode via F11 and sidebar button.
* Replace the default Java input dialog with a custom dark neon `NameInputDialog`.

---

## ✨ Added

### 🖥️ Fullscreen Mode
* **F11 key binding** — toggles between windowed and borderless maximised frame states.
* **Fullscreen sidebar button** — same toggle accessible from the action panel.
* Window position and size are saved and restored on exit from fullscreen.

### 🎨 Render-to-Texture Canvas Scaling
* `GameView.java` now renders internally onto a fixed `800 × 600` `BufferedImage`.
* The image is then scaled to fill the available game panel, making the view resolution-independent and resize-safe.

### 📝 Custom Name Input Dialog
* `NameInputDialog.java` — styled dark neon dialog prompt shown on Game Over.
* Renders a deep dark background, neon border, Segoe UI typography, and a cyan confirm button.
* Replaces the default `JOptionPane.showInputDialog` that broke visual consistency.

### 📊 Live Slider Labels
* Speed slider now shows the active multiplier value next to the track (e.g., `2×`).
* Epsilon slider shows the active exploration rate to two decimal places.

### 🪟 Window Icon & Taskbar Logo
* Application icon set programmatically on the `JFrame` so the snake logo appears in the taskbar and Alt+Tab switcher.

---

## 🔄 Changed

* **Score bug fix** — `GameModel.java` now maps score as `Math.max(0, snake.size() - 1)` instead of raw size, so games start at exactly `0`.
* **Credits** — `CreditsDialog.java` updated with correct contributor name and role; active version label added to the dialog header.
* **Canvas** — `GameView.java` paint output scaled onto the internal `800 × 600` canvas before rendering to the panel.
